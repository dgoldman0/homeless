# Source-History Screen Notes

The earlier adiabatic screen emphasized peak dynamic diagnostics such as `|B_t/B|`, `|B_tt/B|`, and flux. The stricter screen emphasizes the support history sampled over the full operational cycle.

The useful design distinction is:

```math
\text{peak dynamic cost budget}
\quad\text{and}\quad
\text{sampled negative-history budget}
```

In the fixed-`R` `B` cycle, slower ramps improve peak dynamic budgets while extending the duration of the active flare history. In the flare-gated branch, `B` setup/reset happens while the `R` flare is flattened. This moves the core source-history burden into `R`-open, hold, and `R`-close, where it can be budgeted directly.

This is a control and geometry improvement. The next stage applies quantum source-realism and null-contracted sampled-stress gates.
