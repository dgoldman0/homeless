# Framework Alignment Notes

The screening framework decomposes a wormhole-support proposal into support, access, repayment, buffer, matching, transition, and source-realism functions. For the metric family

```math
ds^2=-N(l,t)^2dt^2+B(l,t)^2dl^2+R(l,t)^2d\Omega^2,
```

it assigns:

```math
B(l,t) \rightarrow \text{radial proper-length and support-dilution control},
```

```math
R(l,t) \rightarrow \text{areal access geometry and flare shaping},
```

```math
N(l,t) \rightarrow \text{redshift, timing, matching, and horizon control}.
```

The branch result is consistent with that structure. The B-only protocol passed the dynamic-cleanliness gate but failed the source-history gate because `R=sqrt(1+l^2)` left the areal throat permanently active. The redesign therefore returns to the subsystem allocation layer and activates R as the flare/access-state actuator.

This is not a reversal of the previous warning against R-core breathing. In the branch, R is frozen during the access hold and participates mainly during setup/reset by flattening/restoring flare curvature.
